Unsupervised Machine Learning – Basic Concepts
What is Unsupervised Learning?

Unsupervised Machine Learning is a type of machine learning in which the algorithm learns patterns from unlabeled data — that is, data without predefined categories or outputs.

Unlike supervised learning, there is no "correct answer" provided during training. The model must discover hidden structures on its own.

Unsupervised Learning = Find patterns in data without labels

Common tasks include clustering, dimensionality reduction, and anomaly detection.

  • Clustering: Group similar data points together
  • Dimensionality Reduction: Reduce features while keeping key information (e.g., PCA)
  • Anomaly Detection: Find unusual data points that don't fit any group
What is Clustering?

Clustering is the process of grouping a set of data points into clusters such that points within the same cluster are more similar to each other than to points in other clusters.

It is one of the most popular tasks in unsupervised learning and is used in many real-world applications.

  • Each cluster contains objects that share similar characteristics
  • The algorithm decides the groupings without human supervision
  • The number of clusters (K) is usually defined in advance
Clustering = Automatically grouping similar data points together
What is K-Means?

K-Means is one of the most widely used clustering algorithms. The letter K represents the number of clusters you want to find in the data.

The algorithm partitions the dataset into K groups, where each data point belongs to the cluster with the nearest centroid (center).

  • You define K before running the algorithm
  • Each cluster is represented by its centroid
  • Every data point is assigned to exactly one cluster
  • The goal is to minimize the total distance between points and their cluster centroid
K-Means = Partition data into K clusters by minimizing intra-cluster distance

Steps of the K-Means algorithm:

  1. Choose K (number of clusters)
  2. Initialize K centroids randomly
  3. Assign each data point to the nearest centroid
  4. Recalculate centroids as the mean of all assigned points
  5. Repeat steps 3–4 until the centroids no longer change (convergence)
Centroids

A centroid is the center of a cluster. It represents the average position of all data points assigned to that cluster.

In a 2D space, if a cluster contains the points (2,4), (4,6) and (6,8), the centroid is calculated as:

Centroid = ( (2+4+6)/3 , (4+6+8)/3 ) = ( 4.0 , 6.0 )

Centroids are recalculated at every iteration of the K-Means algorithm. As more iterations run, the centroids move closer to the true center of each cluster until they stabilize.

  • Initial centroids are usually selected randomly
  • They move with each iteration to better represent the cluster
  • When centroids stop moving, the algorithm has converged
Distance Metrics – Euclidean Distance

To assign each data point to the nearest centroid, K-Means uses a distance metric. The most common metric is Euclidean Distance.

The Euclidean distance between two points A(x₁, y₁) and B(x₂, y₂) in 2D is:

d(A, B) = √( (x₂ − x₁)² + (y₂ − y₁)² )

For higher dimensions (n features), it generalizes to:

d(A, B) = √( Σ (aᵢ − bᵢ)² ) for i = 1 to n

Example: Distance between point P(1, 2) and centroid C(4, 6):

d = √( (4−1)² + (6−2)² ) = √( 9 + 16 ) = √25 = 5.0
  • Each data point is assigned to the centroid with the smallest Euclidean distance
  • Other distance metrics exist: Manhattan, Minkowski, Cosine similarity
  • Euclidean distance is the default for K-Means with numerical data
Iterative Process

K-Means is an iterative algorithm: it repeats two steps — assignment and update — until the model converges.

IterationStepAction
1InitializationPlace K centroids randomly in the data space
2AssignmentAssign each point to the nearest centroid
3UpdateRecalculate centroids as the mean of assigned points
4+RepeatReassign and recalculate until centroids don't move
FinalConvergenceCentroids are stable — algorithm stops

Each iteration reduces the total within-cluster distance, making the clusters tighter and more meaningful.

Variance and Convergence

Variance (also called inertia or WCSS – Within-Cluster Sum of Squares) measures how spread out the points are within each cluster:

WCSS = Σ Σ d(xᵢ, Cₖ)²

The goal of K-Means is to minimize WCSS. A lower WCSS means points are closer to their centroids — a better clustering result.

Convergence occurs when the centroids no longer move between iterations — meaning the cluster assignments have stabilized.

  • WCSS decreases with every iteration
  • The algorithm always converges, but may reach a local minimum (not always the global best)
  • Running K-Means multiple times with different initializations helps avoid poor local minima
  • The Elbow Method uses WCSS to choose the optimal K: plot WCSS vs K and look for the "elbow" where improvement slows
Lower WCSS = Tighter, better-defined clusters
Real-World Applications
Customer Segmentation

Retailers and e-commerce platforms use K-Means to group customers by purchasing behavior, demographics, or browsing habits — enabling personalized marketing campaigns.

Document Clustering

Search engines and news platforms cluster articles or documents with similar topics together, helping users discover related content.

Image Compression

K-Means can reduce the number of colors in an image by replacing all colors with the K most representative ones, significantly reducing file size.

Anomaly Detection

Points that don't fit well into any cluster (high distance to centroid) may indicate anomalies — useful in network security and fraud detection.

Medical Imaging

Hospitals use clustering to segment regions in MRI or CT scans, helping identify tumors or tissue types automatically.

Urban Planning

City planners cluster geographic data to identify service zones, optimize traffic flow, or locate infrastructure like hospitals and schools.

Summary

Unsupervised learning allows machines to discover hidden patterns without labeled data. K-Means is one of the simplest and most effective algorithms for clustering.

Key concepts:

  • Unsupervised No labeled outputs required
  • Clustering Group similar data points
  • K-Means Iterative centroid-based clustering
  • Centroid Mean position of a cluster
  • Euclidean Distance Default similarity measure
  • WCSS Metric to minimize — measures cluster tightness
  • Convergence When centroids stop moving
